# 算法概述

## 定义

- 算法是对特定问题求解步骤的描述
- 它是指令的有限序列
- 每一条指令表示一个或多个操作

## 特性

- 有穷性
  - 必须有有穷个步骤
  - 每一个步骤都要在有穷的时间内完成
- 确定性
  - 每一条指令必须有明确的含义，不能产生二义性，
  - 算法只有唯一条执行路径，即相同输入会有相同输出
- 可行性
  - 一个算法必须是可行的
  - 算法的操作必须是已实现的操作的有限次来实现
- 输入
  - 0个或多个 输入
- 输出
  - 1个或多个输出
  - 输出量与输出量有某些特定关系

## 要求

- 正确性
  - 程序不包含语法错误
  - 程序对部分输入输出正确结果
  - 程序对对带有苛刻条件的输入，输出正确结果
  - 程序对任何满足条件的输入，输出正确结果
- 可读性
  - 方便开发者阅读和理解
- 健壮性
  - 对非法输入能够返回对应的错误CODE值，而不是直接终止程序
- 效率与低存储量
  - 对同一个问题，有多种算法可以解决。要选取执行时间较少（效率高），存储量小的算法

## 算法效率的度量

- [时间复杂度>](/notebook/docs/computer/code-design/data-structure/time-complexity)
  - 算法所需存储时间的度量
- 空间复杂度
  - 算法所需存储空间的度量
  - 如果输入数据所占空间只取决于问题本身，和算法无关，只需要分析 除了输入和程序外的，“额外空间”。
  - 如果数据所占空间，与算法有关，也需要分析输入和程序的空间。
  - 如果 “额外空间”相对于输入量是常数，则称为“原地工作”

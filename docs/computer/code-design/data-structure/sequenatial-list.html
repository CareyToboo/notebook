线性表-顺序存储

线性表的使用顺序存储实现，一般称为顺序表。本节的代码内容是非常简单的，除了掌握代码实现以外，还需要了解C语言指针，内存分配与释放，以及内存如何扩容等知识。

知识点​

  • 数据元素逻辑相邻物理存储也相邻，使用元素在列表中的下标表示逻辑相邻关系。
  • 顺序表初始化时需要申请一块完整的地址空间，用于存放数据元素。
    • 由于空间是提前申请的，因此所存放的元素数量是固定的。
    • 当插入元素时，需要注意内存空间溢出的问题。（当插入前，发现即将溢出，可重新申请地址空间，然后再插入数据）
  • 由于每一个数据元素使用的空间是固定的。如果已知某一个位置数据元素的首地址与数据元素的占用空间大小，能够计算出，列表中其他元素的地址。
  已知：

    单个数据元素占用空间：   c = sizeof(ElemType)
    顺序表首元素内存首地址为：LOC(a1)
  
  求：第i个元素，ai的内存首地址是多少？

    LOC(ai) = LOC(a1) + (i-1) * c

总结
  1. 可随机获取某一个元素，查询效率高
  2. 插入或者删除时，需要移动大量元素，修改效率低
  3. 存储结构简单，无需存储额外信息，空间利用率高。
  4. 需要注意内存溢出风险。

重点操作​

在某位置插入元素​

  • 从列表最后一个元素开始，逐个往后移一个位置。

  • 循环的终止条件是：insertPos - 1，一直要移动到插入位置的前一个。

  • 空出插入位置，放入需要插入的新元素。

  • 插入操作的时间复杂度为 O(n)O(n)O(n)

  • 插入元素时，注意额外处理数据溢出逻辑

  • 以下为核心代码，点击查看可运行代码

// 核心代码
// insetPost插入位置，newNode 插入元素
for (int i = L.length - 1; i >= insertPos - 1; i--) { // 从列表最后一个元素开始 到 插入位置。
  L.list[i+1] = L.list[i]; // 逐步往后移一个位
}
L.list[insertPos - 1] = newNode; // 插入位置，放入需要插入的新元素。

删除某位置元素​

  • 删除元素图示

  • 删除时需要先处理将要移除的元素，因为移动后会丢失对应数据。

  • 从移除位置开始，逐个往前移动一个位置

  • 删除操作的时间复杂度为 O(n)O(n)O(n)

  • 以下为核心代码，点击查看可运行代码

// deletePos 删除位置
for (int i = deletePos; i < L.length - 1; i--) {
  L.list[i-1] = L.list[i];
}
